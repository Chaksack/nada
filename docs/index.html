Features

Comprehensive code analysis for Go projects

Static Code Analysis

Deep AST-based inspection of Go syntax trees with comprehensive rule engine

Security Scanning

Detects vulnerabilities including SQL injection, hardcoded secrets, and unsafe practices

Bug Detection

Identifies potential runtime issues, logic errors, and missing error handling

Complexity Analysis

Measures cyclomatic complexity and identifies overly complex functions

Quality Metrics

Comprehensive scoring system with grades A-F and quality trend analysis

Multiple Formats

CLI tool with console output, JSON export, and REST API interface

Installation

Get started with Nada in seconds

Install via Go

go install github.com/chaksack/nada@latest

Requires Go 1.21 or later. The binary will be installed to $GOPATH/bin/nada.

Run Nada in a containerized environment without local installation.

Build from Source

git clone https://github.com/chaksack/nada.git
cd nada
make build

The binary will be built to build/nada.

Usage

Learn how to use Nada effectively

Basic Analysis

# Analyze current directory
nada analyze .

# Analyze specific project
nada analyze /path/to/project

# Export to JSON
nada analyze . --output report.json

Advanced Options

# Include test files
nada analyze . --include-tests

# Verbose output
nada analyze . --verbose

# Exclude files
nada analyze . --exclude='*_test.go,vendor/*'

# With coverage profile
nada analyze . --coverage=coverage.out

Git Integration

# Analyze only staged changes
nada analyze . --diff=staged

# Analyze changes since main
nada analyze . --diff=origin/main

# Analyze unstaged changes
nada analyze . --diff=unstaged

Command Reference
